## Быстрый старт

Установка и запуск за 5 команд:

```bash
# 1. Установка
./install.sh

# 2. Создать домен
mtpx domain add ya.ru

# 3. Создать пользователя (секреты создадутся автоматически)
mtpx user add alice

# 4. Запустить прокси
mtpx apply

# 5. Получить ссылку для подключения
mtpx user link alice
```

## Полный сценарий с нуля

### Шаг 1. Установка

```bash
cd /root/telegaproxy
bash install.sh
```

**Что делает:**
- Проверяет зависимости (docker, openssl, xxd, curl)
- Устанавливает Docker, если отсутствует
- Выставляет права на файлы
- Создаёт symlink `mtpx` в `~/.local/bin/`
- Инициализирует структуру проекта (config/, state/)

### Шаг 2. Добавление доменов

Домены — это серверы, под которые прокси будет маскироваться. Один домен = один Docker-контейнер.

```bash
# Первый домен
mtpx domain add ya.ru

# Второй домен
mtpx domain add google.com

# Третий домен
mtpx domain add cloudflare.com
```

**Что происходит:**
- Создаётся контейнер `mtproto-ya-ru` на порту 443
- Генерируется системный секрет (т.к. пользователей ещё нет)
- Контейнер запускается с конфигом `config/proxy-ya-ru.py`

> **Важно:** без пользователей контейнеры запускаются с пустым списком секретов и не будут принимать подключения. Рекомендуется сначала создать пользователя, потом домены.

**Рекомендуемый порядок:**

```bash
# 1. Создать пользователя
mtpx user add alice

# 2. Добавить домены (секреты создадутся автоматически для alice)
mtpx domain add ya.ru
mtpx domain add google.com
```

### Шаг 3. Создание пользователей

Каждый пользователь получает уникальный секрет для каждого домена.

```bash
# Создать пользователя
mtpx user add alice

# Создать второго пользователя
mtpx user add bob

# Создать с комментарием
mtpx user add charlie "клиент из Telegram"
```

**Что происходит:**
- Запись в `state/users.csv`
- Для **каждого существующего домена** генерируется уникальный Fake TLS секрет
- Конфиги прокси обновляются, контейнеры перезапускаются

### Шаг 4. Получение ссылок

```bash
# Все ссылки пользователя (все домены)
mtpx user link alice

# Ссылка для конкретного домена
mtpx user link alice ya.ru

# Все ссылки всех пользователей (все домены)
mtpx domain links
```

**Пример вывода:**
```
ya.ru:
  tg://proxy?server=94.228.121.38&port=443&secret=ee79612e7275a1b2c3d4e5

google.com:
  tg://proxy?server=94.228.121.38&port=443&secret=ee676f6f676cf6g7h8i9j0
```

### Шаг 5. Раздача ссылок

Отправляете ссылку пользователю. В Telegram (мобильном/десктопном) клик по ссылке автоматически добавит прокси.

- **alice** получает ссылку → подключается → трафик маскируется под HTTPS
- **bob** получает другую ссылку → подключается → трафик маскируется под HTTPS
- Секреты разные — можно отозвать доступ одному, не трогая другого

## Управление пользователями

### Добавить пользователя

```bash
mtpx user add <имя>
```

Автоматически создаёт секреты для всех доменов и перезапускает контейнеры.

### Посмотреть список

```bash
mtpx user list
```

```
┌──────┬──────────────────┬────────────┬──────────┬──────────┐
│ ID   │ Username         │ Created    │ Status   │ Domains  │
├──────┼──────────────────┼────────────┼──────────┼──────────┤
│ u_.. │ alice            │ 2026-04-09 │ active   │ 2        │
│ u_.. │ bob              │ 2026-04-09 │ active   │ 2        │
└──────┴──────────────────┴────────────┴──────────┴──────────┘
```

### Получить ссылки

```bash
# Все домены
mtpx user link alice

# Конкретный домен
mtpx user link alice ya.ru
```

### Отозвать доступ (без удаления)

```bash
mtpx user revoke alice
```

Секреты аннулируются, контейнеры перезапускаются без секретов alice. Пользователь не сможет подключиться, но запись сохраняется.

### Перегенерировать секреты

```bash
mtpx user rotate alice
```

Отзывает старые секреты, создаёт новые. Полезно если секрет утекл.

### Удалить пользователя

```bash
mtpx user remove alice
```

Секреты удаляются, контейнеры перезапускаются. Запись в users.csv получает статус `revoked`.

## Управление доменами

### Добавить домен

```bash
mtpx domain add <домен>
```

**Что происходит:**
1. Генерируется уникальный секрет для **каждого активного пользователя**
2. Создаётся конфиг `config/proxy-<домен>.py`
3. Запускается контейнер `mtproto-<домен>`
4. Если пользователей нет — создаётся системный секрет (пустой USERS)

### Удалить домен

```bash
mtpx domain remove ya.ru
```

**Что происходит:**
1. Контейнер останавливается и удаляется
2. Все секреты для этого домена удаляются
3. Конфиг `config/proxy-ya-ru.py` удаляется

### Перезапустить домен

```bash
mtpx domain restart ya.ru
```

### Остановить / запустить

```bash
mtpx domain stop ya.ru
mtpx domain start ya.ru
```

### Посмотреть логи

```bash
# Последние 20 строк
mtpx domain logs ya.ru

# Последние 50 строк
mtpx domain logs ya.ru 50
```

### Список доменов

```bash
mtpx domain list
```


## Миграция с v2

Если вы обновлялись с v2 (где секреты были без привязки к пользователям), старые секреты нужно мигрировать.

### Автоматическая миграция

```bash
mtpx init
```

При инициализации система автоматически обнаруживает старый формат и добавляет колонку `user_id`.

### Ручная миграция

```bash
mtpx migrate
```

### Полная очистка и создание с нуля

Если миграция не нужна — очистить и начать заново:

```bash
# Удалить все секреты
mtpx secrets clear

# Удалить все домены (контейнеры)
mtpx domain remove ya.ru
mtpx domain remove google.com

# Создать пользователя (секреты создадутся при добавлении доменов)
mtpx user add alice

# Создать домены (секреты для alice)
mtpx domain add ya.ru
mtpx domain add google.com
```

## Частые задачи

### Добавить нового пользователя к существующим доменам

```bash
mtpx user add charlie
mtpx user link charlie
```

Это всё. Секреты создадутся автоматически, контейнеры перезапустятся.

### Добавить новый домен для существующих пользователей

```bash
mtpx domain add duckduckgo.com
```

Секреты создадутся для всех пользователей (alice, bob, charlie).

### Забрать доступ у пользователя

```bash
# Временно (можно восстановить)
mtpx user revoke alice

# Навсегда
mtpx user remove alice
```

### Все пользователи потеряли доступ

```bash
# Посмотреть что происходит
mtpx status
mtpx doctor

# Перезапустить все контейнеры
mtpx apply

# Или конкретный домен
mtpx domain restart ya.ru
```

### Обновить код на сервере

```bash
cd /root/telegaproxy
git pull
chmod +x mtpx

# Если были изменения в структуре данных
mtpx migrate

# Перезапустить
mtpx apply
```

### Проверить состояние

```bash
mtpx status
```

```
║    🟢 google.com         running  port=443
║
╚═════════════════════════════════════════════╝
```

### Полная диагностика

```bash
mtpx doctor
```

Проверяет:
- ✅ Зависимости (docker, openssl, xxd, curl)
- ✅ Структуру файлов
- ✅ Docker-контейнеры
- ✅ Права файлов (должны быть 600)

### Детальная информация о контейнере

```bash
# Все контейнеры
mtpx inspect

# Конкретный домен
mtpx inspect ya.ru
```

### Эвристический мониторинг

```bash
mtpx monitor
```

Показывает по каждому домену: статус, порт, ошибки в логах, uptime.

## Справочник команд

### Основные

| Команда | Описание |
|---------|----------|
| `mtpx init` | Инициализация проекта |
| `mtpx apply [domain]` | Запустить все домены (или один) |
| `mtpx status` | Сводка состояния |
| `mtpx doctor` | Диагностика |
| `mtpx inspect [domain]` | Детали контейнера |
| `mtpx monitor` | Эвристический мониторинг |

### Домены

| Команда | Описание |
|---------|----------|
| `mtpx domain add <domain>` | Создать домен + контейнер + секреты |
| `mtpx domain remove <domain>` | Удалить домен + контейнер |
| `mtpx domain list` | Все домены |
| `mtpx domain links` | Все ссылки для всех доменов |
| `mtpx domain restart <domain>` | Перезапустить |
| `mtpx domain stop <domain>` | Остановить |
| `mtpx domain start <domain>` | Запустить |
| `mtpx domain logs <domain> [n]` | Логи |

### Пользователи

| Команда | Описание |
|---------|----------|
| `mtpx user add <username>` | Создать пользователя |
| `mtpx user remove <username>` | Удалить пользователя |
| `mtpx user list` | Список пользователей |
| `mtpx user link <username> [domain]` | Ссылка (все или конкретный домен) |
| `mtpx user revoke <username>` | Отозвать доступ |
| `mtpx user rotate <username>` | Перегенерировать секреты |

### Операции

| Команда | Описание |
|---------|----------|
| `mtpx secrets clear` | Удалить все секреты |
| `mtpx migrate` | Миграция v2→v3 |
| `mtpx stop` | Остановить все |
| `mtpx restart` | Перезапустить все |

## Архитектура

```
ya.ru       → mtproto-ya-ru       → alice=ee7961..., bob=ee7962...
google.com  → mtproto-google-com  → alice=ee676f..., bob=ee6770...
```

- **Образ:** alexbers/mtprotoproxy (Python, multi-secret)
- **Конфиг:** `config/proxy-<domain>.py` монтируется в контейнер
- **Пользователи:** `state/users.csv`
- **Секреты:** `state/secrets.csv` (привязка: user + domain)
- **Обновление:** при добавлении/удалении пользователя контейнеры перезапускаются автоматически

## Обратная совместимость

```bash
# Старый способ (работает)

# Новый способ (рекомендуется)

./mtpx user add alice
./mtpx user link alice
```
